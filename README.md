# DropBid — Dutch Auction Platform

**DropBid** — платформа аукціонів зі зниженням ціни (Dutch auction) з унікальною моделлю монетизації.  
Продавець завжди отримує ринкову ціну. Покупець отримує геймифіковані торги з можливістю зекономити.

---

## Як працює аукціон

### Механіка (крок за кроком)

```
1. Продавець виставляє товар за ринковою ціною (наприклад, iPhone = €1000)
   └── При створенні ціна перевіряється на актуальність

2. Покупець платить вхідний внесок = 1% від стартової ціни (€10)
   ├── Ціна лоту знижується на суму внеску: €1000 → €990
   ├── На 5 секунд показується: поточна ціна + кількість учасників
   └── Протягом цих 5 секунд нові входи заморожені

3. Будь-який учасник може:
   ├── Закрити аукціон — доплатити поточну ціну і забрати товар
   └── Переглянути ціну в будь-який момент за €1 (5 секунд інформації)

4. Аукціон закривається коли:
   ├── Хтось доплачує поточну ціну (закриває аукціон)
   └── Або ціна доходить до €0 (останній учасник отримує товар)

5. Оплати збираються на ескроу-рахунку аукціону
   └── Продавець отримує гроші після підтвердження отримання товару
```

### Фінансова модель

```
Ескроу = Σ(всі вхідні внески) + фінальна ціна закриття = ЗАВЖДИ = стартова ціна

Приклад: iPhone €1000
  50 учасників × €10 = €500 (вхідні)
  Переможець закриває на €500
  Ескроу = €1000 → продавцю ✅

  Або 100 учасників × €10 = €1000
  Ціна = €0, останній отримує безкоштовно
  Ескроу = €1000 → продавцю ✅

Платформа заробляє ТІЛЬКИ на переглядах ціни (€1 за перегляд)
```

### Ролі користувачів

| Роль | Можливості |
|---|---|
| **Buyer** | Участь в аукціонах, перегляд ціни, закриття аукціону |
| **Seller** | Створення аукціонів, верифікація ЄС, отримання коштів |
| **Admin** | Управління платформою, верифікація продавців, аналітика |

## Поточний стан проекту

### ✅ Реалізовано
- Реєстрація та авторизація (buyer / seller / admin)
- Повна бізнес-логіка аукціону (вхід, зниження ціни, закриття)
- 5-секундне вікно заморозки при вході
- Ескроу-система (збір коштів, підтвердження, виплата продавцю)
- Верифікація продавців (документи, ЄС-вимоги)
- Завантаження фото лотів
- Підключення Web3-гаманця (MetaMask)
- AI-асистент з голосовим вводом (OpenAI + Whisper)
- Blockchain-оплати через Polygon (MATIC)
- AUKTO токен — знижки на перегляд ціни

### ⚠️ Потребує доопрацювання
- Немає real-time оновлень (WebSocket/SSE) — інші учасники не бачать зміну ціни
- SQLite → PostgreSQL для продакшну
- Немає валідації ринкової ціни при створенні (заплановано)
- `User.query.get()` → `db.session.get()` (SQLAlchemy 2.0 deprecation)
- Немає тестів (pytest)

### ❌ Відсутній дизайн
- Bootstrap за замовчуванням, немає брендингу
- Головна сторінка — стіна тексту, немає візуальної ієрархії
- Немає онбордингу — пояснення механіки для нових користувачів
- Немає таймерів / прогрес-барів зниження ціни

## Встановлення та запуск

### Вимоги
- Python 3.10+
- pip

### Кроки

```bash
# 1. Клонувати репозиторій
git clone https://github.com/pilipandr770/auction7.git
cd auction7

# 2. Створити та активувати venv
python -m venv venv
venv\Scripts\activate      # Windows
# source venv/bin/activate  # Linux/Mac

# 3. Встановити залежності
pip install -r requirements.txt

# 4. Налаштувати змінні середовища
cp .env.example .env
# Відредагуйте .env — заповніть SECRET_KEY та blockchain-адреси

# 5. Ініціалізувати базу даних (PostgreSQL)
#    DATABASE_URL у .env, напр.: postgresql+psycopg2://postgres:postgres@localhost:5433/dropbid
#    Створити порожню БД, потім застосувати міграції:
flask db upgrade
#    (опціонально) тестові дані:
python seed_data.py

# 6. Запустити
python run.py
# → http://127.0.0.1:5000
```
